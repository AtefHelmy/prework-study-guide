HTML

  • The head section contains metadata, refer to this file for common declarations
  • The body element contains the webpage data the user sees
  • The entire code goes inside of an html element, where you also define the language <html lang="en">
  • The title element goes in the head, and is critical for the page
  • The base element defines the base URL for the page and it must be in the head
  • The link element connets an external resource to the HTML document
  • The script element embeds client-side scripts in the HTML document

CSS

  • The style element is the way that you add css styling to your html elements
  • You can style the entire document with *
  • You can style an element, all of an element tag with element {style-attribute: value;}
  • you can create and define classes inside html elements and then style these elements with .classname {}
  • I tried messing around and centering the unordered list, but ended up having trouble, it would center the text, but not bullet points. To solve this, you have to change the value of list-style-position to inside, default it is outside.
  • I had trouble changing the space between text and bullet points. One outside solution was to put the actual text inside li element within a span element. Now within the span element, can make the position of the text, not the line of the bullet point, but just the text itself position relative and move it to the left and right. Example of how to do so is commented at the bottom of the html code for this page.
  • Every element follows the box model. There is internal padding, a border, and external margins, which along with the actual size of the element, determine the overall width and length of the element
  • We used border to create a border, and input a thickness in pixels, and a color. We used box-shadow to create the box-shadow effect. It can have up to 4 inputs, which are x displacement, y displacement, shadow blurring, and shadow spreading.
  • I thought the top and bottom of the border was too close to the text in the border, so I added padding to the top and bottom of the entire card. In addition, I thought the bottom of title of each card was too close to the first bullet point. I could have done this either via heading or list, but I added a top-margin to lists inside each card.

GIT

  • To make edits, create branch, make local edits, add changes to git commit , then commit those changes to the working local git version, then push those changes to the github repository.
  • Everytime we start a coding session, we first want to start the ssh agent, which is accomplished by first starting the agent with eval "$(ssh-agent-s)", and then adding our computer SSH key with ssh-add
  • git init to initialize a local repository
  • git status to check which branch, and see changes to be commited
  • git add to add a change to the changes to be committed
  • git commit to commit to the local working git repository branch
  • git push origin branchname to push to online github
  • git pull origin branchname to pull copy from github, which is good to do at the start of your coding sessions if there are others working on a branch with you
  • Probably the easiest and best way to make local and online respositories sync is to first make one online via Github, and then to clone that repository using git clone URL/SSH
  • git checkout to change branches, -b to make a new branch
  • git add -A to add ALL changes that can be committed
